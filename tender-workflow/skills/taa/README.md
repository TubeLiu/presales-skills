# Tender Assistant — 招标分析助手

**中文** | [English](./README_EN.md)

> 一条命令，从百页招标文件到可执行的投标策略。

**Tender Assistant** 是一个 **Claude Code** AI Skill，专为售前投标团队打造。它能深度解读招标文件，自动生成结构化的 **招标分析报告（.md）** 和 **投标文件大纲（.docx）**，将原本需要数小时的人工通读、摘录、整理工作压缩到几分钟完成。

## 为什么需要它

| 传统做法 | 使用 Tender Assistant |
|---------|---------------------|
| 手动通读上百页招标文件 | AI 自动提取全部关键信息 |
| 逐条摘录技术要求，容易遗漏 | 7 大模块系统化分析，★号条款零遗漏 |
| 凭经验判断废标风险 | 废标条款逐条列举 + TOP5 风险排序 |
| 评分项凭感觉写 | 每个评分项给出具体拿高分策略 |
| 大纲结构每次从零搭起 | 自动生成 Word 大纲，含章节编号和需求-章节对应表 |

## 核心亮点

**深度分析，不是简单摘要** — 7 大分析模块（M1-M7）覆盖项目概况、技术矩阵、商务条款、评分策略、废标风险、格式清单、撰写指导，每条分析标注原文出处（章节/页码）。

**行业智能识别** — 自动识别政府IT、国企/央企、金融三大行业，加载对应扩展模块（如政采合规、信创要求、金融监管），通用项目使用默认模板。

**可直接用的双文件输出** — Markdown 分析报告供团队内部决策，Word 大纲（.docx）直接作为标书撰写骨架，含中文字体排版和 A4 页面设置。

**需求全覆盖校验** — Part C 需求-章节对应表确保招标文件中所有【必须】条款都有对应大纲章节，防止关键需求遗漏。

## 使用方法

在 Claude Code 中执行：

```bash
# 基本用法（使用默认索引或 AI 模糊评估）
/taa <招标文件路径>

# 指定产品能力说明书（精确评估匹配度）
/taa <招标文件路径> --product <产品能力说明书.xlsx|.md>

# 分析时保存索引到默认位置
/taa <招标文件路径> --product <产品能力说明书.xlsx> --save-index

# 指定厂商名称（必填，可在 /twc setup 配持久值）
/taa <招标文件路径> --vendor "博云"

# 组合使用
/taa <招标文件路径> --product specs/产品能力.md --vendor "博云"

# 仅构建产品能力索引（不执行分析）
/taa --build-index --product <产品能力说明书.xlsx>

# 从环境变量配置的默认文件构建索引
/taa --build-index

# 强制使用 AnythingLLM 作为产品能力来源（不可用则报错）
/taa <招标文件路径> --kb-source anythingllm

# 强制使用本地索引，跳过 AnythingLLM
/taa <招标文件路径> --kb-source local
```

也支持自动触发：上传招标文件后输入"分析标书"、"评估招标文件"、"看看这份招标文件"等指令，Skill 会自动调用。

### 产品能力来源优先级

```
--product 指定文件（最高优先，精确评估）
    ↓ 未指定时
AnythingLLM 语义检索（M2 阶段按需查询，查询缓存机制）
    ↓ 不可用时
本地 YAML 索引（V2 三层 / V1 单文件）
    ↓ 均无时
AI 模糊评估
```

**`--kb-source` 参数**（v2.2.1 新增）:
| 参数值 | 说明 |
|--------|------|
| `auto`（默认） | 先尝试 AnythingLLM，不可用则降级到本地索引 |
| `anythingllm` | 强制使用 AnythingLLM（不可用则报错退出） |
| `local` | 强制使用本地索引，跳过 AnythingLLM 检测 |

**注意**: 若同时使用 `--product` 和 `--kb-source anythingllm`，后者优先（强制使用 AnythingLLM）。

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| `<招标文件>` | 位置参数 | 必选。招标文件路径 |
| `--product` | 文件路径 | 可选。产品能力说明书（Excel/Markdown），用于精确评估产品与招标要求匹配度。**注意**：若同时使用 `--kb-source anythingllm`，此参数将被忽略（强制使用 AnythingLLM） |
| `--vendor` | 字符串 | 必填。厂商名称（可在 /twc setup 配持久值；缺失会报错引导） |
| `--build-index` | 标志 | 可选。仅构建产品能力索引并保存到默认位置，不执行分析 |
| `--save-index` | 标志 | 可选。分析时将索引保存到默认位置（需配合 `--product` 使用） |
| `--anythingllm-workspace` | 字符串 | 可选。指定 AnythingLLM workspace slug（推荐）或名称。未指定时使用配置文件，否则自动取第一个 workspace |
| `--kb-source` | 字符串 | 可选。产品能力来源控制：`auto`（默认，先 AnythingLLM 后本地索引）、`anythingllm`（强制使用 AnythingLLM，不可用则报错）、`local`（强制使用本地索引，跳过 AnythingLLM） |

### 环境变量

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `TAA_DEFAULT_PRODUCT` | 默认产品能力文件路径，用于 `--build-index` 无参数时 | `/data/products/ACP_4.2.0.xlsx` |
| `TAA_ANYTHINGLLM_WS` | 默认 AnythingLLM workspace，优先级低于配置文件和命令行参数 | `product-kb` |

**配置文件**：`~/.config/tender-workflow/config.yaml`（统一配置，通过 `/twc setup` 管理）
```yaml
anythingllm:
  workspace: product-kb              # 全局默认 AnythingLLM workspace
```

Skill 自动执行三个阶段：

```
Phase 0  读取文件 → 自动判断行业类型 → 加载行业模板
Phase 1  7 模块深度分析 → 输出 招标分析报告_[时间戳].md
Phase 2  生成投标大纲 → 输出 投标文件大纲_[时间戳].docx
```

输出文件保存在 `./output/` 目录。

## 分析报告详解

### 基础模块（所有项目）

| 模块 | 内容 | 核心价值 |
|------|------|---------|
| **M1** | 项目概况与投标建议 | 一张表看清项目全貌，给出明确的投/不投建议（强烈建议/建议/谨慎/不建议） |
| **M2** | 技术要求分析矩阵 | 9 类技术要求逐条提取，标注优先级；有 `--product` 时自动填写支持度，否则 AI 模糊评估 |
| **M3** | 商务要求与合同条款 | IP 归属、违约金、分包限制等深度分析，高风险条款标注 `[难点]` |
| **M4** | 评分细则与得分策略 | 每个评分子项给出**具体可执行的拿高分方法**，而非空洞的"突出优势" |
| **M5** | 废标条款与风险排序 | `[废标]` / `[扣分]` 分离标注，TOP5 风险优先级排序 + 应对策略 |
| **M6** | 格式要求 Checklist | 可直接打勾的格式确认清单，含容易遗漏的细节提示 |
| **M7** | 标书撰写指导 | 撰写优先级、关键词清单、评分项写作策略、差异化亮点建议、撰写红线 |

### 行业扩展模块（自动加载）

| 行业 | 扩展模块 | 关注重点 |
|------|---------|---------|
| 政府IT | E1 预算合规 · E2 政采合规 · E3 安全保密 | 资金来源、中小企业政策、等保要求 |
| 国企/央企 | E1 国资监管 · E2 信创要求 · E3 廉政风控 | 国资审批、CPU/OS/数据库型号、回避制度 |
| 金融 | E1 金融监管 · E2 数据安全 · E3 业务连续性 | 银保监/证监会要求、数据分级、RTO/RPO |

## 投标大纲输出

大纲基于分析报告动态生成，包含三部分：

- **Part A — 大纲说明**：结构决策依据（为什么这样组织章节）
- **Part B — 完整大纲**：仅技术部分（1.1-1.11），服务方案整合到末尾
- **Part C — 需求-章节对应表**：每个【必须】条款 → 对应章节编号，确保零遗漏

生成的 `.docx` 文件已预设：
- 中文字体（各级标题统一宋体）
- A4 页面（上下 2.5cm，左右 2.4cm）
- Word 标题样式（支持自动生成目录）

## 分析质量保证

- 每条分析标注原文出处，格式：`（出处：第X章 P.XX）`
- 废标项 `[废标]`、扣分项 `[扣分]`、难点 `[难点]` 精确标注
- 信息缺失填"**文件未明确**"，严禁臆测数字
- M7 撰写指导针对当前项目具体展开，拒绝通用模板内容

## 项目结构

```
tender-assistant/
├── SKILL.md                         # Claude Code Skill 主文件（Phase 0/1/2 完整 prompt）
├── prompts/
│   ├── analysis.yaml                # 7 模块分析框架定义
│   └── outline.yaml                 # 大纲生成框架定义
└── templates/builtin/
    ├── government_it.yaml           # 政府IT行业扩展模块
    ├── state_owned_enterprise.yaml  # 国企/央企行业扩展模块
    ├── finance.yaml                 # 金融行业扩展模块
    └── general.yaml                 # 通用行业（默认）
```
