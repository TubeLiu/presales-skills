# tpl v2.0 模板使用指南

## 概述

tpl v2.0 聚焦于**招标技术规格与评标办法**生成，通过反控标转换将产品功能清单自然地转换为甲方视角的招标要求。行业模板提供差异化的技术规格框架、评标权重和反控标配置。

## 命令格式

```bash
/tpl <产品功能清单> [--project <项目概述>] --template <行业> [--level <级别>] [--no-scoring]
```

## 模板列表

| 模板名称 | 适用场景 | 核心关注点 | 价格权重 | 技术权重 | 指标敏感度 |
|---------|---------|-----------|---------|---------|-----------|
| **government** | 政府机关、事业单位 | 合规性、信创、等保 | 30% | 50% | high |
| **finance** | 银行、保险、证券 | 金融级安全、高可用、监管合规 | 25% | 50% | medium |
| **soe** | 中央企业、国有企业 | 自主可控、稳定性、长期支持 | 30% | 45% | high |
| **enterprise** | 民营企业、外资企业 | 性价比、灵活性、快速交付 | 40% | 40% | low |

> **指标敏感度**：反控标转换时指标调整的激进程度。high = 更积极地避免品牌关联，low = 企业采购灵活度高，可适度宽松。

## 模板结构（v2.0）

每个行业模板 YAML 包含以下顶层配置：

| 配置块 | 用途 |
|--------|------|
| `characteristics` | 行业特征（采购方式、权重、核心关注点） |
| `tech_spec_defaults` | 技术规格默认框架（章节结构、行业专项章节） |
| `scoring_defaults` | 评标办法默认配置（权重、商务评分、技术评分） |
| `anti_control_overrides` | 反控标行业覆盖（允许术语、标准引用、指标敏感度） |

## 四级细致程度

通过 `--level` 控制输出深度：

| 级别 | 技术规格页数 | 功能要求条目 | 压缩比 | 适用场景 |
|------|------------|------------|--------|---------|
| `detailed` | 15-20 页 | 40-60 条 | ~1:1 | 大型项目 >500 万 |
| `standard`（默认） | 8-12 页 | 15-25 条 | ~3:1 | 中型项目 100-500 万 |
| `general` | 4-6 页 | 8-12 条 | ~8:1 | 小型项目 50-100 万 |
| `brief` | 2-3 页 | 5-8 条 | ~15:1 | <50 万或内部参考 |

## 控标点规则

产品功能描述中包含**"控标点"**字样的条目，**必定纳入**最终招标要求：
- 不受 level 压缩比影响，不会被合并或删除
- 仍执行术语中性化（去品牌名），但保留核心技术指标的实质内容
- 评标办法中优先映射为高分值评分项

## 模板详细说明

### government - 政府行业

**适用场景**：政府机关信息化建设、事业单位数字化转型、政务云平台、智慧城市项目

**行业专项章节**（自动注入技术规格）：
- 等保安全要求（三级及以上）
- 国产化/信创要求（国产化率≥70%）

**反控标允许术语**：Kubernetes、CNCF、OCI、CNI、CSI

**使用示例**：
```bash
# 标准技术规格与评标办法
/tpl features.txt --template government

# 详细技术规格，含项目背景
/tpl features.txt --project overview.txt --template government --level detailed

# 使用知识库，只生成技术规格
/tpl --kb --template government --no-scoring
```

---

### finance - 金融行业

**适用场景**：银行核心系统、保险业务平台、证券交易系统、金融科技创新

**行业专项章节**（自动注入技术规格）：
- 金融级高可用架构要求（两地三中心、99.99%可用性）
- 金融级安全要求（加密、审计、数据不出境）
- 监管合规要求（银保监会、人民银行规范）

**反控标允许术语**：Kubernetes、CNCF、OCI、CNI、CSI

**使用示例**：
```bash
# 详细技术规格与评标办法
/tpl features.txt --project overview.txt --template finance --level detailed

# 简略技术规格，内部参考
/tpl features.txt --template finance --level brief
```

---

### soe - 央国企

**适用场景**：中央企业信息化建设、国有企业数字化转型、大型集团IT平台

**行业专项章节**（自动注入技术规格）：
- 自主可控技术路线要求
- 国产化适配要求（CPU/OS/DB/中间件）
- 长期技术支持要求（≥5年）

**反控标允许术语**：Kubernetes、CNCF、OCI、CNI、CSI

**使用示例**：
```bash
# 标准技术规格与评标办法
/tpl features.txt --template soe

# 使用知识库 + 文件组合输入
/tpl features.txt --kb --template soe --level standard
```

---

### enterprise - 通用企业

**适用场景**：民营企业、外资企业、互联网公司、创新型企业

**行业专项章节**（自动注入技术规格）：
- 灵活部署要求（公有云/私有云/混合云）
- 快速交付要求（≤3个月、敏捷开发）
- 成本优化要求（资源利用率、按需扩展、TCO）

**反控标允许术语**：Kubernetes、CNCF、OCI、CNI、CSI、Docker

**使用示例**：
```bash
# 标准技术规格与评标办法
/tpl features.txt --template enterprise

# 只生成技术规格，不要评标办法
/tpl features.txt --template enterprise --no-scoring

# 简略输出，内部参考
/tpl --kb --template enterprise --level brief --no-scoring
```

## 模板选择建议

### 按行业选择

| 行业 | 推荐模板 | 理由 |
|------|---------|------|
| 政府机关、事业单位 | government | 严格合规、信创要求 |
| 银行、保险、证券 | finance | 金融级安全、高可用 |
| 央企、国企 | soe | 自主可控、长期支持 |
| 民企、外企、互联网 | enterprise | 灵活部署、快速交付 |

### 按项目特征选择

| 项目特征 | 推荐模板 | 理由 |
|---------|---------|------|
| 强调国产化 | government 或 soe | 信创要求、国产化适配 |
| 强调安全性 | finance | 金融级安全、灾备 |
| 强调稳定性 | soe | 长期支持、稳定优先 |
| 强调性价比 | enterprise | 成本优化、灵活部署 |

## 注意事项

1. **模板选择**：根据招标方行业和需求选择合适的模板
2. **Level 选择**：根据项目预算规模选择合适的细致程度
3. **控标点标记**：在产品功能清单中，关键差异化功能描述里加入"控标点"字样
4. **合规审核**：生成后建议通过 `/trv --type tender_doc --focus compliance` 进行深度审核
5. **反控标检查**：输出文件已包含自动反控标自检，但建议人工复核
